# Experience Direction Workflow

Use this workflow when the creator asks for broad gameplay design, progression, systems, modpack concepts, or a project direction.

## Goal

Define the intended player experience before specialists produce content.

## Steps

1. Interpret creator intent.
   - Fantasy.
   - Player goal.
   - Scope.
   - Target Minecraft version and loader.
   - Multiplayer and performance constraints.
   - Preference for custom content, existing mods, or hybrid.

2. Choose project mode.
   - Single feature with clear output -> Focused Mod Mode.
   - Multiple new mechanics or progression stages -> Original Design Mode.
   - Existing mod reuse or pack composition -> Modpack Author Mode.
   - Existing mods plus custom content -> Hybrid Mode.

3. Define player journey.
   - Early game.
   - Mid game.
   - Late game.
   - Endgame.

4. Identify required systems.
   - Combat.
   - Progression.
   - Economy and Loot.
   - Tech.
   - Magic.
   - Worldgen.
   - Quest and Guide.
   - Balance.
   - Modpack Integration.

5. Produce a contract outline.
   - Experience contract.
   - System contracts.
   - Feature contracts.
   - Specialist outputs.
   - QA gates.

6. Dispatch production.
   - Send system work to design modules.
   - Send scoped work to specialists.
   - Preserve contract and QA requirements.

## Guardrails

- Do not start broad requests by generating an item, mob, texture, or block.
- Do not let one specialist accidentally define the whole game loop.
- Do not over-plan Focused Mod Mode.
- Prefer existing mods for modpack authoring when they satisfy the experience.
